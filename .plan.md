# 解构赋值实现计划

## 总览

实现三种解构上下文：声明解构、函数参数解构、默认值。

解构本质是**语法糖 lowering**：将 `swc_ast::Pat` 模式展开为一系列 `GetProp`/`GetElem` + `StoreVar` IR 指令。IR 层、WASM 后端、运行时**无需新增任何指令或 builtin**——现有基础设施完全足够。

## Phase 1：核心 Pattern Lowering

新增 `Lowerer::lower_destructure_pattern(pat, src_val, block) -> Result<()>`。

按 `Pat` 变体分发：
- `Pat::Ident`：复用现有 StoreVar 逻辑
- `Pat::Object`：遍历 props，每个 property 做 GetProp + 递归，Rest 用 ObjectRest builtin
- `Pat::Array`：遍历 elems，每个 element 做 GetElem(index_const)，默认值用 IteratorFrom/Next/Value/Done + Phi
- `Pat::Assign`：StrictEq 比较 src_val 与 undefined，Branch + lower_expr(default) + Phi

辅助函数 `lower_default_value_check(value, default_expr, block) -> Result<ValueId>` 内联 Pat::Assign 逻辑。

## Phase 2：修改声明 lowering

修改 `lower_var_decl`：
- 对 `Pat::Array`/`Pat::Object` 不再报错，改为调用 `lower_destructure_pattern`
- 对没有 init 的 let/var 声明：用 `Const(Undefined)` 作为 src_val
- 对 const 无 init：报错

## Phase 3：修改函数参数 lowering

修改 `lower_fn_decl` / `lower_fn_expr` / `lower_arrow_expr` 的参数处理：
- 每个参数生成临时变量承载参数值
- 对 `Pat::Assign { left, right }`：做 default value check，然后递归 `lower_destructure_pattern(left, ...)`
- 对 `Pat::Object`/`Pat::Array`：调用 `lower_destructure_pattern`
- 对 `Pat::Ident`：直接 StoreVar
- param_ir_names 依然包含所有参数临时变量 + $env + $this

## Phase 4：预扫描（predeclare）

检查并修改 `predeclare_block_stmts` / `predeclare_stmt`：当前预扫描可能不处理 `Pat::Array`/`Pat::Object` 中的嵌套绑定。需要递归遍历 pattern 将所有 Ident 绑定注册到作用域树。

## Phase 5：测试

新增 8 个 happy 测试 + 1 个 error 测试，更新 IR 快照，运行验证。
